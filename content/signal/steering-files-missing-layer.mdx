## The problem with prompts

- Prompts are per-session and per-person; knowledge evaporates.
- Inconsistent output across engineers and across time.

## What a steering file is

- A reusable instruction set encoding architecture decisions, standards,
  security policy, and domain rules.
- The format agents consume automatically (e.g. `CLAUDE.md`, rules files, MCP).

## Anatomy of a good steering file

- Scope and precedence.
- Hard constraints vs. preferences.
- Examples and counter-examples.

## Operating the layer

- Versioning, review, and ownership.
- How steering files compound into the knowledge flywheel.

## A starter template

A minimal steering file you can adopt this week, with notes on what to add next.
